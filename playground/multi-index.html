🗂️ MultiIndex

tsb's MultiIndex is a multi-level hierarchical index — the TypeScript equivalent of pandas.MultiIndex. Where a normal Index labels each row with a single value, a MultiIndex labels each row with a tuple of values, one per level.

Internally the data is stored efficiently as an array of per-level distinct label arrays (levels) plus parallel arrays of integer codes that index into those levels. This representation is compact when the same label combinations repeat across many rows.

Creating a MultiIndex

fromArrays — parallel label arrays

import { MultiIndex } from "tsb";

// Two parallel arrays — one per level
const mi = MultiIndex.fromArrays(
  [["a", "a", "b", "b"], [1, 2, 1, 2]],
  ["letter", "number"],   // optional level names
);

mi.size;      // 4
mi.nlevels;   // 2
mi.names;     // ["letter", "number"]
mi.at(0);     // ["a", 1]
mi.at(-1);    // ["b", 2]
MultiIndex([("a", 1), ("a", 2), ("b", 1), ("b", 2)], names=['letter', 'number'])

fromTuples — array of label tuples

const mi2 = MultiIndex.fromTuples(
  [["US", "CA"], ["US", "TX"], ["EU", "DE"]],
  ["region", "country"],
);

mi2.size;   // 3
mi2.at(1);  // ["US", "TX"]

fromProduct — Cartesian product

// All combinations of letters × numbers
const mi3 = MultiIndex.fromProduct(
  [["a", "b", "c"], [1, 2]],
  ["letter", "number"],
);

mi3.size;     // 6 (3 × 2)
mi3.toArray();
// [["a",1], ["a",2], ["b",1], ["b",2], ["c",1], ["c",2]]

Accessing Level Values

getLevelValues(level) returns a flat Index<Label> of all labels at the given level — the labels in their original (non-unique) order, one per row. 

const mi = MultiIndex.fromArrays(
  [["a", "a", "b", "b"], [1, 2, 1, 2]],
  ["letter", "number"],
);

mi.getLevelValues(0).toArray();   // ["a", "a", "b", "b"]
mi.getLevelValues(1).toArray();   // [1, 2, 1, 2]
mi.getLevelValues("letter").toArray();  // same — resolve by name

The levels property gives you the distinct labels per level: 

mi.levels[0].toArray();  // ["a", "b"]  (sorted unique values)
mi.levels[1].toArray();  // [1, 2]

Lookup by Key

getLoc(key) finds the positional index (or indices) of a tuple. Partial keys (shorter than nlevels) match all rows whose first levels match. 

const mi = MultiIndex.fromTuples(
  [["a", 1], ["a", 2], ["b", 1], ["b", 2]],
  ["letter", "number"],
);

mi.getLoc(["a", 1]);  // 0   ← single match, returns number
mi.getLoc(["a"]);     // [0, 1]  ← partial key, returns number[]
mi.contains(["b", 2]);  // true
mi.contains(["z", 9]);  // false

Manipulation

droplevel

const mi = MultiIndex.fromTuples(
  [["a", 1, "x"], ["b", 2, "y"]],
  ["p", "q", "r"],
);

// Drop one level — returns Index

swaplevel

const mi = MultiIndex.fromTuples(
  [["a", 1], ["b", 2]],
  ["letter", "number"],
);

const swapped = mi.swaplevel(0, 1);
swapped.at(0);    // [1, "a"]
swapped.names;    // ["number", "letter"]

setNames / setLevels / setCodes

const mi = MultiIndex.fromTuples([["a", 1], ["b", 2]]);

const renamed = mi.setNames(["letter", "number"]);
renamed.names;  // ["letter", "number"]

// Replace the distinct values for level 0
const updated = mi.setLevels(["A", "B"], 0);
updated.levels[0].toArray();  // ["A", "B"]

Sorting

const mi = MultiIndex.fromTuples([
  ["b", 2], ["a", 1], ["a", 2], ["b", 1],
]);

const sorted = mi.sortValues();  // ascending (default)
sorted.toArray();
// [["a",1], ["a",2], ["b",1], ["b",2]]

// Per-level direction: first level ascending, second descending
const sorted2 = mi.sortValues([true, false]);
sorted2.toArray();
// [["a",2], ["a",1], ["b",2], ["b",1]]

Set Operations

const m1 = MultiIndex.fromTuples([["a", 1], ["b", 2]]);
const m2 = MultiIndex.fromTuples([["c", 3], ["d", 4]]);

const combined = m1.append(m2);
combined.size;   // 4
combined.at(2);  // ["c", 3]

m1.equals(MultiIndex.fromTuples([["a", 1], ["b", 2]]));  // true

Conversion

const mi = MultiIndex.fromTuples(
  [["a", 1], ["b", 2]],
  ["letter", "number"],
);

// All tuples as an array
mi.toArray();   // [["a",1], ["b",2]]

// As a plain object for DataFrame construction
mi.toRecord();  // { letter: ["a", "b"], number: [1, 2] }

// Deduplication
const dup = MultiIndex.fromTuples([["a", 1], ["b", 2], ["a", 1]]);
dup.nunique();           // 2
dup.dropDuplicates().toArray();  // [["a",1], ["b",2]]

Iteration

const mi = MultiIndex.fromTuples([["a", 1], ["b", 2], ["c", 3]]);

for (const [letter, num] of mi) {
  console.log(`${String(letter)}-${String(num)}`);
}
// "a-1"
// "b-2"
// "c-3"

Inspecting the Structure

const mi = MultiIndex.fromProduct(
  [["Q1","Q2"], [2023, 2024]],
  ["quarter", "year"],
);

mi.size;     // 4
mi.nlevels;  // 2
mi.shape;    // [4, 2]
mi.ndim;     // 2 (always)
mi.isUnique;            // true
mi.isMonotonicIncreasing;  // true (product is already sorted)

console.log(mi.toString());
// MultiIndex([(Q1, 2023), (Q1, 2024), (Q2, 2023), (Q2, 2024)],
//            names=['quarter', 'year'])
Tip: Use MultiIndex.fromProduct when building panel datasets (e.g. entity × time). Use fromTuples when you already have the row labels. Use fromArrays when your data arrives as separate per-level columns.
