Series.map()

Map values using a function, Record, Series, or ES6 Map — mirrors pandas.Series.map.

Function mapper

Apply a function (value, index, pos) => U to every element.

import { Series } from "tsb";

const prices = new Series({ data: [10, 20, 30], name: "price" });

// Double each value
const doubled = prices.map(v => (v as number) * 2);
// doubled → [20, 40, 60]

// Format as currency string
const formatted = prices.map(v => `$${v}`);
// formatted → ["$10", "$20", "$30"]

Record / dict mapper

Look up each value (stringified) in a plain JS object. Missing keys produce null.

import { Series } from "tsb";

const codes = new Series({ data: [1, 2, 3, 9] });
const result = codes.map({ "1": "low", "2": "med", "3": "high" });
// result → ["low", "med", "high", null]  (9 not in dict)

Series mapper

Look up each value by label in another Series. The values of this are treated as index labels into the mapper Series. Missing labels produce null.

import { Index, Series } from "tsb";

const cats = new Series({ data: ["dog", "cat", "fish"] });

const legCounts = new Series({
  data: [4, 4, 0],
  index: new Index(["dog", "cat", "fish"]),
});

const result = cats.map(legCounts);
// result → [4, 4, 0]

ES6 Map mapper

Use a native Map<Scalar, U> for non-string keys (numbers, booleans, null, etc.).

import { Series } from "tsb";

const flags = new Series({ data: [true, false, true] });
const m = new Map([[true, "yes"], [false, "no"]]);
const result = flags.map(m);
// result → ["yes", "no", "yes"]

naAction: "ignore"

Pass { naAction: "ignore" } to preserve existing null/NaN values without looking them up.

import { Series } from "tsb";

const s = new Series({ data: [1, null, 2] });
const result = s.map({ "1": "one", "2": "two" }, { naAction: "ignore" });
// result → ["one", null, "two"]
// Without naAction, null would be looked up as "null" → null anyway,
// but for NaN the distinction can matter.
