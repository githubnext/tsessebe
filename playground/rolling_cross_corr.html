🔁 Rolling Cross-Correlation

crossCorrrollingCrossCorr Measures how similar two signals are when one is shifted relative to the other at various lags.

📖 Concept

Cross-correlation at lag l is the Pearson correlation of x[i] with y[i − l] across all valid pairs.

  • lag = 0 — ordinary Pearson correlation
  • lag > 0 — y leads x (positive shift)
  • lag < 0 — x leads y (negative shift)

rollingCrossCorr computes this inside a trailing window, returning a DataFrame with one column per lag.

💡 crossCorr — full series

import { Series, crossCorr } from "tsb";

const x = new Series({ data: [1, 2, 3, 4, 5] });
const y = new Series({ data: [0, 1, 2, 3, 4] }); // y = x shifted by 1

const result = crossCorr(x, y, { lags: [-1, 0, 1] });
// lag_neg1 → ?
// lag_0    → 1   (x and y are perfectly linearly related)
// lag_1    → 1   (pairs x[1..4] with y[0..3] = [1..4] with [0..3])

💡 rollingCrossCorr — windowed

import { Series, rollingCrossCorr } from "tsb";

const x = new Series({ data: [1, 2, 3, 4, 5, 6, 7] });
const y = new Series({ data: [0, 1, 2, 3, 4, 5, 6] });

const df = rollingCrossCorr(x, y, 4, { lags: [-1, 0, 1] });
// Returns a DataFrame with 7 rows and columns: lag_neg1, lag_0, lag_1
// Rows 0..2 are null (window not yet full)
// Row 3 onward: Pearson corr of x[i-3..i] vs y shifted by each lag

📚 API Reference

// Full-series cross-correlation
crossCorr(
  x: Series<Scalar>,
  y: Series<Scalar>,
  options?: {
    lags?: number[];      // explicit lags (overrides maxLag)
    maxLag?: number;      // default: min(n-1, 20)
    minPeriods?: number;  // default: 2
  }
): Series<Scalar>  // indexed by lag label (e.g. "lag_0", "lag_neg2")

// Rolling windowed cross-correlation
rollingCrossCorr(
  x: Series<Scalar>,
  y: Series<Scalar>,
  window: number,
  options?: {
    lags?: number[];      // explicit lags
    maxLag?: number;      // default: min(window-1, 5)
    minPeriods?: number;  // default: 2
  }
): DataFrame  // rows=time, columns=lag labels
