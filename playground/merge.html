merge

Database-style joins between DataFrames — inner, left, right, outer.

How merge works

merge(left, right, options) performs an SQL-style join of two DataFrames. You specify which columns to join on (the key), and how to handle rows that don't match via the how parameter:

inner — only rows whose key appears in both tables
left — all rows from the left table; null where no right match
right — all rows from the right table; null where no left match
outer — all rows from both tables; null where no match

Interactive example — try different join types

Change how to "inner", "left", "right", or "outer" and click Run.

left_on / right_on — different key names per side

When the key column has a different name in each table, use left_on and right_on.

Suffixes — resolving column name conflicts

When both tables have a non-key column with the same name, tsb appends configurable suffixes (default "_x" / "_y").

Many-to-many joins

When a key value appears multiple times on both sides, merge produces the Cartesian product of matching rows (just like SQL).
